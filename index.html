Mendelbase.jl

MendelBase contains the base functions of OpenMendel.

Download this project as a .zip file Download this project as a tar.gz file

Overview

MendelBase is a utility package that contains the base functions of OpenMendel. It provides the information needed to process keywords and read data from external files. MendelBase is necessary to run any analysis package, and you should install MendelBase first, before you install the other packages.

Installation

Within Julia, use the package manager to install MendelBase:

Pkg.clone("git@github.com:OpenMendel/MendelBase.jl.git")

This package supports Julia v0.4.

Input Files

To make your genetic data available to OpenMendel, the various input files must be in a format that OpenMendel can read. To provide OpenMendel with the information it needs to perform its analysis you must prepare the following files. Examples input files can be found in the docs subfolders of the different analysis packages. (An analysis option won't always need every file type.)

  • Control File: Specifies the names of your data input and output files and any optional parameters (keywords) for the analysis. (For a complete list of keywords, see Keywords Table below).
  • Locus File: Names and describes the genetic loci in your data.
  • Pedigree File: Gives information about your individuals, such as name, sex, family structure, and ancestry.
  • Phenotype File: Lists the available phenotypes.
  • SNP Definition File: Defines your SNPs with information such as SNP name, chromosome, position, allele names, allele frequencies. This may be a PLINK .bim format file.
  • SNP Data File: Holds the genotypes for your data set. Must be a standard binary PLINK .bed file in SNP major format. If you have a SNP data file you must have a SNP definition file.

Control files

The format of the Control file is

Keyword = Keyword_Value(s)

Below is an example Control file to run GWAS (text after "#" is ignored and treated as comments):

#
# Input and Output files.
#
plink_input_basename = gwas 1 data
output_file = gwas 1 Output.txt
manhattan_plot_file = ManPlotOutput_1.png
#
# Analysis parameters for GWAS option.
#
regression = linear
regression_formula = Trait ~ Sex

In the example above, there are five keywords. The first three specify input and output files: plink_input_basename, output_file, manhattan_plot_file. The last two specify analysis parameters: regression and regression_formula. The text after the '=' are the keyword values. For keywords that can take a set of values, list the values separated by commas.

General Data File Format

All data files are in text file format with one exception: if you have dense SNP data, the SNP data should be in binary files in PLINK .bed file format. OpenMENDEL will also accept PLINK format .fam and .bim. All non-PLINK OpenMendel data files list data in columns, with a header row with field names for each column. The column need not be in any fixed order. Many column headers have required, case sensitive names. The column separators can be any single character, specified in the control file with the field_separator. The the default separator is comma. If a data element contains the field separator (e.g. comma) it must be enclosed in double quotes, or it will be read as a new field. Missing data is blank ("") or "NA". The PLINK file missing data symbols are "0" or "-9".

Locus File

The Locus file defines your genetic markers unless you are working with a dense SNP map, in which case you must use a SNP Definition. The Locus file must have a header row with the field names (in any order): Locus, Allele, Chromosome, Frequency. If your samples come from different populations, instead of one column labeled Frequency, you may name the populations in the control file with the keyword populations, and give an allele frequency for each population. The population frequency field names are the population names specified in the control file. The population frequencies need not sum to 1, but will be normalized to 1 in the analysis. You may also include locus position information with optional fields Morgans, FemaleMorgans and MaleMorgans, and Basepairs. For each locus there will be one line per locus allele.

Example

Locus,Allele,Chromosome,European,African,Chinese
ABO,A,9,0.27,0.18,0.19
ABO,B,9,0.06,0.11,0.17
ABO,O,9,NA,0.71,0.64
Rh,D,1,0.61,0.81,1
Rh,d,1,0.39,0.19,0
SNP,1,9,NA,NA,NA
SNP,2,9,NA,NA,NA
XSNP,1,X,0.3,0.4,0.5
XSNP,2,X,0.7,0.6,0.5

Pedigree File

A Pedigree file is always required. It can be in PLINK FAM file format, in which case it must have the extension .fam, or it can have the format of a required header rows with the following fieldnames (in any order): Pedigree,Person,Mother,Father,Sex,Twin, and optional columns such as population ancestries. The fractional ancestries need not sum to 1, but will be normalized to 1 in the analysis.

Example

Pedigree,Person,Mother,Father,Sex,Twin,BirthYear,Politician,European,African,Chinese,ABO,Rh,Xg,XSNP,HtMeters
Clinton,Bill,,,male,0,1946.0,true,1.0,0.0,0.0,A,+,+,2/2,1.88
Clinton,Hillary,,,female,0,1947.0,true,1.0,0.0,0.0,O,-,-,1/1,NA
Clinton,Chelsea,Hillary,Bill,female,0,1980.0,false,NA,NA,NA,A,+,-,1/2,NA
Obama,Barack,,,male,0,1961.0,true,0.5,0.5,0.0,O,+,NA,1/1,1.85
Obama,Michelle,,,female,0,1964.0,false,0.3,0.7,0.0,B,+,NA,2/2,NA
Obama,Malia,Michelle,Barack,female,0,1998.0,false,NA,NA,NA,O,-,+,1/2,NA
Obama,Sasha,Michelle,Barack,female,0,2001.0,false,NA,NA,NA,B,+,+,1/2,NA
